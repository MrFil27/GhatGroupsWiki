Comandi ChatGroups

Sottocomandi

I comandi di ChatGroups hanno come prefisso di default "chatggorup", tuttavia è stato aggiunto un alias che facilità e velocizza l'uso dei comandi. Puoi anche usare direttamente il comando /cg help per ottenere un supporto più rapido direttamente nel tuo server.

  • /cg help (permessi: cg.admin, cg.use): fornisce una lista di tutti i comandi citati in questa guida

  • /cg create <name> <owner> (permessi: cg.admin): creerà una nuova chat, aggiornerà di conseguenza anche il file di configurazione.

  • /cg addmember <group> <player> (permessi: cg.admin, cg.use): aggiunge un membro alla chat indicata, aggiornerà di conseguenza anche il file di configurazione; il nuovo membro potrà parlare in tale chat.

  • /cg addowner <group> <player> (permessi: cg.admin, cg.owner): aggiunge un owner alla chat indicata, aggiornerà di conseguenza anche il file di configurazione; il nuovo owner potrà gestire e parlare in tale chat.

  • /cg removemember <group> <player> (permessi: cg.admin, cg.use): rimuove un membro dalla chat indicata, aggiornerà di conseguenza anche il file di configurazione; il nuovo membro non potrà più parlare in tale chat.

  • /cg removeowner <group> <player> (permessi: cg.admin, cg.owner): rimuove un owner dalla chat indicata, aggiornerà di conseguenza anche il file di configurazione; il nuovo owner non potrà più gestire e parlare in tale chat.

  • /cg toggle <group> (permessi: nessuno): permette al giocatore di attivare la chat indicata; dal momento di esecuzione tutti i messaggi del player verranno inviato in tale chat, per disattivare l'esecuzione rieseguire nuovamente il comando.

  • /cg reload (permessi: cg.admin): Ricarica il file di configurazione e applicherà le modifiche effettaute in esso.

Messaggio rapido

È possibile scrivere in una chat senza la necessità di dover eseguire il sottocomando toggle:

  • /message <group> <message> o /msg <group> <message> (permessi: nessuno)

Azioni

Il plugin supporta 3 tipi di azioni riportate precedentemente dalla configurazione:

  • /azione <string>, /action <string> o /a <string> (permessi: nesusno): simula un'azione eseguita dal player a una distanza definita dalla configurazione.

  • /urlo <string> o /scream <string> (permessi: nesusno): invia un messaggio a una distanza definita dalla configurazione; per mantenere la realisticità del plugin, impostare un raggio maggiore rispetto a chat-radius.

  • /bisbiglio <string>, /bisbiglia <string> o /whisper <string> (permessi: nesusno): invia un messaggio a una distanza definita dalla configurazione; per mantenere la realisticità del plugin, impostare un raggio minore rispetto a chat-radius.
